Release Notes_

Changelog for the Cybat Intelligence Engine. Each release covers engine logic improvements, platform features, and infrastructure changes.

Version 1.0.0

LATEST April 2026

Ingestion Pipeline

  • GKE Ingestion Layer: Pub/Sub bridge and Fluent Bit run as a sidecar pair on GKE. The bridge continuously pulls per-organization subscriptions (soc-logs-{orgId}-sub), injects routing metadata, and forwards records to Fluent Bit's HTTP input. Fluent Bit applies a Lua context filter to extract source IPs before forwarding to the SOC agent via an OIDC-authenticated egress proxy — all within the same pod.
  • Multi-tenant Subscription Discovery: The bridge automatically discovers all active per-org Pub/Sub subscriptions on a 60-second refresh cycle, requiring no manual configuration when new organizations onboard.
  • Zero-ETL Latency: End-to-end pipeline lag from log emission to Cloud Run SOC agent is consistently under 7 seconds in steady state.

AI Threat Analysis

  • Vertex AI Gemini 2.5 Flash Lite: The SOC agent sends per-IP log batches to Gemini 2.5 Flash Lite via Vertex AI. The model returns a structured verdict (threat level + explanation) with automatic retry on transient failures.
  • 24-Hour Verdict Cache: IP verdicts are cached in Firestore using HMAC-SHA256 hashed IP keys for 24 hours. Repeat offenders are blocked instantly without consuming AI quota.
  • 1-Hour History Window: The agent considers the last 60 minutes of activity per IP to give Gemini full behavioral context, improving detection of slow-burn and distributed attacks.
  • Duplicate Enforcement Guard: A per-IP enforcement lock prevents redundant firewall or Cloud Armor rule creation if multiple batches arrive concurrently for the same IP.

Enforcement

  • Dual Enforcement Layer: VPC Firewall deny rules are attempted first (Layer 3/4). If the VPC network cannot be resolved from the log's subnet labels, the engine falls back to a Cloud Armor IP deny rule (Layer 7) on the Load Balancer.
  • Per-Project Scoping: Enforcement is applied to the specific GCP project identified in the log's resource.labels.project_id or logName field — not globally, preserving blast-radius isolation in multi-project organizations.

Platform & Dashboard

  • Intelligence Dashboard: Real-time threat command center at intelligence.cybat.co. Features a 3D Threat Globe, live incident feed, and full incident detail panels with Gemini analysis and enforcement history.
  • Google SSO: All access is gated by Google Identity. No separate credentials are required — users sign in with their existing Google Workspace account.
  • Slack Alerting: Every high and medium incident dispatches a structured Slack notification to the organization's configured webhook, including source IP, geo location, threat explanation, and enforcement outcome.
  • Firestore Incident Store: All incidents, AI verdicts, log evidence, and enforcement records are persisted to Firestore under orgs/{orgId}/incidents, enabling full audit trails and historical review.

Marketplace & Billing

  • GCP Marketplace Integration: Available on the Google Cloud Marketplace with Starter, Professional, and Enterprise tiers. Billing is metered through GCP Service Control — charges appear on your existing GCP bill with no separate invoice.
  • Entitlement-Based Activation: The SOC agent validates your active Marketplace entitlement via Service Control services.check before processing any logs. Expired or canceled subscriptions halt enforcement within one billing cycle.
