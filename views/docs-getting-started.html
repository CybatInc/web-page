Getting Started_

Welcome to the Cybat Intelligence Engine. Activating your autonomous AI-driven security operations through the Google Cloud Marketplace is designed to be completely frictionless. Follow these steps to secure your environment in minutes.

Prerequisites

Before subscribing, confirm your GCP environment meets the following requirements. The engine can be activated without all items, but detection quality improves significantly with each one in place.

Required

  • ✓ Active GCP project with billing enabled
  • ✓ Cloud Logging enabled on your project
  • ✓ At least one internet-facing workload (GCE, GKE, or Cloud Run)
  • ✓ VPC network configured
  • ✓ Slack workspace + Incoming Webhook URL for alert delivery

Strongly Recommended

  • + Cloud HTTP(S) Load Balancer with access logging enabled — unlocks full L7 threat analysis
  • + Cloud Armor security policy attached to the Load Balancer — enables edge IP blocking
  • + Structured JSON logs with httpRequest.remoteIp or jsonPayload.ip fields

For a complete list of supported log types and required fields, see the Security Specs page.

Step 1: Subscribe via Google Cloud Marketplace

  • Navigate to the Google Cloud Console.
  • Go to the Marketplace and search for Cybat Intelligence Engine.
  • Land on the Cybat product page and click the Subscribe button.
  • Select your billing account, review the pricing tier, and accept the Terms of Service.
  • Once the subscription is active, click Register with Cybat to be redirected to our secure provisioning portal.

Step 2: Authenticate with Google Workspace (SSO)

Cybat relies strictly on Google Identity for secure, centralized access.

  • On the Cybat portal landing page, click Sign in with Google.
  • Select your corporate Google Workspace account.
  • Upon first login, your Organization profile will be automatically provisioned based on your domain.

Step 3: Configure Project Access & IAM Permissions

To maintain our zero-retention, non-intrusive architecture, Cybat analyzes your logs asynchronously via Pub/Sub.

  • Run the Provisioning Script: Upon logging in, you will be provided a one-click Google Cloud Shell command. This will automatically deploy the minimal infrastructure required.
  • What the script does: Creates a Log Router Sink, deploys a dedicated Pub/Sub Topic, and grants the Cybat Engine service account the strict roles required: logging.viewer and compute.networkViewer.

The Cybat Intelligence Engine is now actively monitoring your cloud surface!
