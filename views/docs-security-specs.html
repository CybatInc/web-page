Critical Security Specs_

Permissions (IAM)

The Engine operates under a principle of least privilege. Requires explicit authorization for: logging.viewer, compute.networkViewer, compute.securityAdmin (for Cloud Armor enforcement), compute.networkAdmin (for VPC Firewall enforcement).

Data Sovereignty

Zero-retention architecture ensures data remains strictly within the user's GCP project boundaries. No logs or sensitive payload data are persisted to Cybat infrastructure.

Authentication

Platform access relies exclusively on Single Sign-On (SSO) via Google Identity integration, seamlessly authorizing users against their existing Workspace/Cloud Identity directory.

Client Requirements_

Minimum infrastructure and log configuration needed for the engine to detect threats effectively.

Infrastructure Prerequisites

Required

  • ✓ GCP project with Cloud Logging enabled
  • ✓ At least one internet-facing workload generating HTTP logs (GCE, GKE, Cloud Run)
  • ✓ VPC network — required for Layer 3/4 firewall enforcement
  • ✓ Slack workspace for real-time alert delivery

Recommended

  • + Cloud HTTP(S) Load Balancer — enables full L7 log analysis and Cloud Armor integration
  • + Cloud Armor security policy — enables edge-level IP blocking (Layer 7)
  • + GKE Autopilot or Fluent Bit agent — for structured log forwarding from containerized workloads

Supported Log Sources

Cloud HTTP(S) Load Balancer

Full L7 Coverage

The primary and most effective log source. Provides httpRequest.remoteIp, httpRequest.requestUrl, and HTTP status codes. Enables SQLi, XSS, credential stuffing, and DDoS detection. Requires Cloud Logging to be enabled on the backend service.

Cloud Armor Logs

Signature + L7 Context

Pre-rule evaluation logs from Cloud Armor. Provides attack signature context alongside source IP and matched OWASP rule IDs. Works in combination with Load Balancer logs for enriched analysis. Cloud Armor policy must be attached to the Load Balancer.

GKE / Application Logs

Structured JSON Required

Logs emitted from containerized workloads via Fluent Bit or the GKE logging agent. Must include a structured jsonPayload.ip field. Optionally include jsonPayload.request for URL-level analysis.

Custom Application Logs

Partial Coverage

Any structured JSON log emitted to Cloud Logging. Detection quality depends on the fields present. The engine will extract threat signals as long as a source IP is resolvable.

VPC Flow Logs / Raw TCP

L3/L4 Only

VPC Flow Logs are supported but provide network-layer signals only (port scans, volumetric traffic). Application-layer attacks (SQLi, XSS, injection) cannot be detected without HTTP-level log fields. Non-HTTP protocols are not analyzed.

Required & Recommended Log Fields

Field Status Purpose
httpRequest.remoteIp
or jsonPayload.ip		 Required Source IP for grouping and blocking. Without this field, no analysis or enforcement is possible.
resource.labels.project_id
or logName		 Required GCP project identification for multi-tenant routing, billing, and enforcement targeting.
httpRequest.requestUrl
or request		 Recommended Enables URL-pattern analysis for SQLi, XSS, and path traversal detection. Significantly improves threat classification accuracy.
httpRequest.status Recommended HTTP status code. 4xx/5xx responses are used as primary signal for attack pattern clustering.
resource.labels.subnetwork_name Optional Used to auto-discover the VPC network for targeted firewall rule creation. Without it, enforcement falls back to Cloud Armor.

Limitations & Out-of-Scope

  • ✗ Non-GCP environments: The engine is native to Google Cloud. AWS, Azure, and on-premise log sources are not supported.
  • ✗ Unstructured / plaintext logs: Logs must be structured JSON to be parseable. Free-text log lines cannot be analyzed.
  • ✗ Encrypted payload inspection: The engine operates on log metadata. It does not decrypt or inspect TLS payload bodies.
  • ✗ Non-HTTP protocols: Raw TCP/UDP, DNS, and SMTP attack patterns are outside the current detection scope.
  • ✗ Internal traffic only: Workloads without an external IP or Load Balancer will not generate the log fields needed for effective threat detection.
